# 05_UI_Reference｜UI 与交互参考说明

## 1. 设计定位

风格关键词：

- B 端工具
- AI 设计生产力
- 清爽
- 科技感
- 高效
- 低学习成本

## 2. 页面布局

建议桌面端使用左右布局：

```text
┌───────────────────────────────┬───────────────────────────────┐
│ 左侧配置区                     │ 右侧结果预览区                 │
│                               │                               │
│ 素材上传                      │ 750×750 卡片                  │
│ 文案输入                      │ 530×706 卡片                  │
│ 颜色配置                      │ 750×400 卡片                  │
│ Doodle 配置                   │ 750×750（二楼）卡片            │
│ 生成按钮                      │ 342×514 卡片                  │
│                               │ Doodle 卡片                   │
└───────────────────────────────┴───────────────────────────────┘
```

## 3. 左侧配置区

### 3.1 素材上传

分成两个卡片：

#### 营销图素材

- 背景图上传
- 主视觉产品图上传
- Logo 上传

#### Doodle 素材

- Doodle PNG 上传
- 单行 / 双行切换
- 单行文案
- 双行第一行
- 双行第二行

## 4. 文案区

文案字段不要暴露复杂业务名称，统一使用：

- 文案一
- 文案二
- 文案三
- 文案四
- 文案五
- 文案六
- 按钮文字一
- 按钮文字二
- 标签文案

输入框 placeholder 使用 JSON 示例：

- 出行抽一抽 / 领假日券包
- 主题六个字内
- 利益点文案九个字内
- 立即查看
- 立即抢购
- 标签文案区

## 5. 颜色区

每一项包含：

- 颜色预览块
- HEX 输入框
- Color Picker

建议字段：

- 主标题颜色
- 副标题颜色
- 按钮底色
- 按钮文字颜色
- 标签底色
- 标签文字颜色
- Doodle 文字颜色

## 6. 结果卡片

每张结果卡片包含：

```text
尺寸名称
图片预览
下载 PNG
下载 JPG
```

Doodle 卡片额外提示：

```text
PNG 保留透明背景，JPG 不支持透明背景。
```

## 7. 操作按钮

主按钮：

```text
一键生成
```

次按钮：

```text
重置默认值
清空输入
下载全部 ZIP
```

## 8. Loading 状态

生成中显示：

```text
正在生成 6 张宣传图，请稍候…
```

每张结果卡片可以显示 skeleton 占位。

## 9. 错误状态

常见错误提示：

```text
请先上传背景图。
```

```text
请先上传主视觉产品图。
```

```text
请上传 Doodle PNG 素材。
```

```text
生成失败，请稍后重试。
```

```text
图片过大，请压缩后重新上传。
```

## 10. 响应式

MVP 可以优先桌面端。

如果做移动端：

- 配置区在上
- 结果区在下
- 结果卡片单列展示

## 11. 推荐组件

使用 shadcn/ui：

- Card
- Input
- Button
- Tabs
- RadioGroup
- Select
- Dialog
- Progress
- Toast

## 12. 默认色彩建议

页面 UI 不要过度花哨，让用户生成的图片成为视觉焦点。

建议：

- 背景：#F7F8FA
- 卡片：#FFFFFF
- 主色：#2563EB
- 文字主色：#111827
- 次级文字：#6B7280
- 分割线：#E5E7EB
